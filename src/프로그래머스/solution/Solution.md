## 풀이에 대한 설명
문자열을 앞에서부터 확인하면서
( 는 +1, ) 는 -1로 count 값을 관리하였다.

중간에 count가 음수가 되면 닫는 괄호가 더 많은 경우이므로 false를 반환한다.
모든 문자열을 확인한 뒤 count가 0이면 올바른 괄호로 판단한다.


## 풀어나가는 과정에서 시행 착오

처음에는 스택을 사용하려고 했지만, 괄호 종류가 하나라서 불필요하게 복잡해졌다.

또한, 마지막에 count를 확인하지 않아 "(((" 같은 경우를 놓쳤고,
중간에 음수 체크를 하지 않아 ")()" 같은 경우도 제대로 처리하지 못했다.

-> 이를 해결하기 위해 스택 대신 count 변수를 사용하여 괄호 개수를 관리하고, 반복문 중간에 count가 음수가 되면 바로 false를 반환하도록 수정하였다.
